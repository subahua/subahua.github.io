String类学习记录

1：从JDK8源码中分析

1:String类被final修饰,因此String不能被子类继承。

2:String类实现Serializable接口,表示String类可以序列化。

3:String类实现Comparable接口,表示String类在进行比较的时候最好不要用”==”。

4:String类实现了CharSequence接口。

5:String类数据存储在char[]数组中,char[]数组被final修饰,表示String对象是不可变类型,一旦初始化,值就不可更改。

2：字符串常量池

这行代码从表面上看创建了一个对象,实际上是两个。当new 一个String对象是,java虚拟机会先在字符串常量池中查找有没有”王胖子爱学习“这个对象，如果有，就不会再在字符串常量池创建“王胖子爱学习”这个对象，直接在堆中创建一个“王胖子爱学习”的对象，然后在将堆中“王胖子爱学习”这个对象的地址值返回赋值给s1。反之，如果没有，会先在字符串常量池中创建“王胖子爱学习”这个对象，再在堆中创建一个“王胖子爱学习”这个对象，然后将堆中“王胖子爱学习”这个对象返回赋值给s1。

一般来说，创建字符串对象，都会使用双引号，而不会通过new来创建。在这行代码中，这种方式创建字符串对象时，Java虚拟机会先在字符串常量池中查找有没有“王胖子爱学习”这个对象，如果有，则不创建任何对象，直接将字符串常量池中“王胖子爱学习”这个对象地址返回赋值给s2，如果没有，在字符串常量池中创建“王胖子爱学习”这个对象，然后将这个对象的地址返回赋值给s2。

3：intern

知识点1：使用双引号声明的字符串对象会保存在字符串常量池中。

知识点2：使用new关键字创建的字符串对象会先在字符串常量池中寻找这个对象，如果没有找到，就创建一个，然后再在堆中创建这个对象；如果找到了，就直接在堆中创建这个对象。

这行代码中s3没有使用双引号声明”王胖子爱学习”这个对象，因此如果想要将“王胖子爱学习”这个对象在字符串常量池中创建一个的话，可以使用intern方法。

String s4 = new String("王胖子") + new String("爱学习");这行代码,java虚拟机会先在字符串常量池中创建两个对象”王胖子”和”爱学习”,再在堆中创建两个匿名对象”王胖子”和”爱学习”，一个“王胖子爱学习”对象，s4引用的就是堆中”王胖子爱学习“的这个对象。

String s5 = s4.intern(); s4对象调用intern()方法，该方法会先在字符串常量池中查询“王胖子爱学习”这个对象是否存在，此时，是不存在的，但是堆中已经存在此对象了，因此字符串常量池会报错堆中“王胖子爱学习”这个对象的引用，因此s5的地址与s4的地址相同。
